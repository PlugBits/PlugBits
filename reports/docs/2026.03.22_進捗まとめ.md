PlugBits Reports の Cloud Run renderer について、前スレの続きです。

まずはこのまとめを前提に読んでください。
[cloudrun_renderer_current_summary_2026-03-23.md]

現状の要点だけ先に書くと：

- Worker / Queue / consumer / remote handoff までは通っています
- dedup 問題は改善済みです
- renderer は単独 build / Cloud Run deploy 成功済みです
- ただし、Cloud Run service で `POST /internal/render` に対して 202 を返し、その後 detached async で重い PDF 生成を続ける方式が安定していない可能性が高いです
- Queue が悪いのではなく、Cloud Run service + detached async の方式自体が怪しい段階です
- 見た目修正より先に、アーキテクチャ判断をしたいです

新スレでは、次の論点を中心に議論したいです。

1. 今の Cloud Run service + detached async を続けるべきか
2. Cloud Run Jobs に切り替えるべきか
3. Cloud Tasks + Cloud Run service にするべきか
4. PlugBits Reports の帳票生成に最も相性の良い非同期方式は何か
5. 最小変更で次に進むならどの案がよいか

お願いしたいこと：
- まず 3案（現行方式 / Cloud Run Jobs / Cloud Tasks）の比較をしてください
- そのうえで、私の今の実装状況を踏まえた推奨案を1つに絞ってください
- さらに、その案での最小実装ステップを CODEX 指示レベルまで具体化してください

背景として重要なのは、
- すでにかなりの時間を使っているので、これ以上「その場しのぎの延命」で沼りたくない
- ただし、いきなり全面再設計も避けたい
- まずは安定して job が完走する方式に乗せたい
- その後に見た目やテンプレ互換を詰めたい

この前提で、かなり率直に判断してください。
