📌 estimate_v1 doc-meta 座標リセット問題 完全解消ログ
■ 発生していた問題
Admin Tunerで以下要素を移動しても：
doc_no_label
doc_no
date_label
issue_date
複製・保存・PDFプレビュー時に
座標が既定値（例：y=86 / 106）へ戻る
Admin Tunerが正にならない
Canvas / PDF / Base が一致しない
という重大な整合性崩れが発生。
■ 根本原因
1️⃣ doc-metaラベルがslotSchema管理外だった
doc_no_label に slotIdが無かった
date_label はslotIdがあるが、slotSchemaに存在しなかった
結果：
normalize / 補完処理で
「正規メンバーではない要素」として扱われ
presetレイアウトで再生成される
＝ 座標が巻き戻る
2️⃣ slotSchema→elements補完処理が座標を上書きしていた
既存要素があるにも関わらず、
slotSchema基準で再配置
document meta特別処理
duplicate時の再構築
が発生していた可能性。
■ 実施した修正
✅ ① slotSchemaにdoc-metaラベルを正規追加
headerへ追加：
{
  "slotId": "doc_no_label",
  "label": "見積番号ラベル",
  "kind": "text"
},
{
  "slotId": "date_label",
  "label": "発行日ラベル",
  "kind": "text"
}
✅ ② elements側を完全正規化
doc_no_label に slotId: "doc_no_label" を付与
4要素すべてがslotSchemaの正規メンバーに
✅ ③ 座標上書き禁止ルールを確立
既存要素が存在する場合：
x,y,width,height は絶対に変更しない
補完は「欠落フィールドのみ」
これでAdmin Tunerが唯一の真実に。
✅ ④ デバッグログで経路を完全可視化
追加ログ：
[DBG_SAVE_DOCMETA]
[DBG_FETCH_DOCMETA]
[DBG_RENDER_DOCMETA]
確認結果：
保存 → 取得 → render まで
doc-metaの x/y が完全一致。
PDF側ログ：
[DBG_PDF_DOCMETA_SRC] source: 'elements'
＝ PDFも elements を信じて描画。
■ 最終状態
Admin Tuner = Base = UserTemplate = Canvas = PDF
doc-metaは完全に通常要素扱い
special layout 経路は排除
presetは構造定義のみ
■ 重要な設計原則（今後の鉄則）
🔒 原則1
Admin Tunerが絶対正
🔒 原則2
slotSchemaは「構造定義」であり
座標を持たない
🔒 原則3
elementsが描画の唯一のソース
🔒 原則4
既存elementのx/yを補完処理で上書きしない

doc-metaを「特別扱い」した瞬間に整合性が崩れる。
帳票エンジンにおいて：

全要素を同じパイプラインで処理することが
最も安定する。
